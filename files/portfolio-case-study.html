Project Overview

Serenity Quest reimagines meditation apps by applying gamification principles to create sustained user engagement. Many meditation apps suffer from high drop-off rates after initial enthusiasm wanes. This project explores how game mechanics can motivate consistent mindfulness practice without compromising the peaceful nature of meditation.

The Challenge

How might we encourage consistent meditation practice among college students who struggle with maintaining wellness habits?

The Solution

A gamified meditation platform with XP progression, streak tracking, milestone unlocking, and calming aesthetics that balances motivation with mindfulness.

User Research Insights

Through interviews with 15 college students, three major themes emerged:

  • Lack of Immediate Feedback Students felt meditation benefits were too abstract and long-term. They wanted visible progress indicators.
  • Motivation Drop-Off Initial enthusiasm faded after 1-2 weeks without external accountability or rewards.
  • Time Constraints Long sessions (20+ minutes) were perceived as impossible to fit into busy schedules.

2. Persistent Progress Tracking

Using localStorage, the system maintains user progress across sessions, creating a sense of long-term investment and achievement. All progress persists even when users close their browser or navigate away.

  • XP System Every completed session awards experience points (50-150 XP based on duration). XP is displayed in header with comma formatting for large numbers. Users start at 0 XP and can track their total growth.
  • Gems Currency Secondary reward system (2-8 gems per session) could unlock future premium features like new meditation themes or ambient sounds.
  • Smart Streak Tracking Checks if user meditated yesterday to increment streak, resets after missed days, and doesn't double-count same-day sessions. Starts at 0 and builds with consistency.
  • Achievement Persistence Completed achievements are stored and badges turn green permanently. System tracks total sessions, time of day, and XP milestones to automatically award achievements.
  • Level Progression Users begin at Level 0 (Beginner) with an empty progress circle, creating immediate satisfaction when first progress appears. Progress ring fills gradually as users earn XP.

3. Gamified Journey Path

Five milestone stages (Beginner ‚Üí Practitioner ‚Üí Seeker ‚Üí Adept ‚Üí Master) create clear progression goals. Users start at Beginner with all other milestones locked and grayed out. As users earn XP and complete sessions, milestones unlock sequentially, building anticipation and providing long-term motivation.

4. Achievement System

Badges reward specific behaviors and turn green when completed. All achievements are visible from the start with full-color icons on white backgrounds. Upon completion, badges transform with a green background and white text to clearly indicate progress.

Technical Implementation

Front-end Architecture

Built with vanilla HTML, CSS, and JavaScript for performance and accessibility. No frameworks ensures fast load times and simplicity.

Data Persistence

localStorage API manages user progress, allowing offline functionality and instant load times without backend infrastructure.

Responsive Design

Mobile-first approach with breakpoints at 768px ensures excellent experience on all devices from phones to desktop.

Accessibility

WCAG 2.1 AA compliant color contrast, keyboard navigation support, and semantic HTML for screen readers.

Code Highlights

  • Circular Timer Animation SVG stroke-dashoffset calculation creates smooth countdown visualization synchronized with JavaScript setInterval.
  • Streak Logic Date comparison algorithm checks if last session was yesterday, today, or earlier to maintain accurate streak counting.
  • Achievement Tracking System Automatic detection of achievement conditions (first session, 7-day streak, time-based badges, milestone completions) with green visual state changes and localStorage persistence.
  • Confetti Celebration Dynamically generated DOM elements with randomized positions, colors, and animations celebrate session completion.
  • Progress State Management All user data (XP, gems, streaks, achievements, total sessions) managed through localStorage with real-time UI updates and persistent state across sessions.

Key Learnings & Reflections

  • Balancing Gamification with Mindfulness The biggest challenge was incorporating motivating game mechanics without creating anxiety or competitive stress. Soft colors, gentle animations, and personal (not social) progress tracking solved this.
  • Importance of Quick Wins 5-minute sessions were crucial. Users appreciated low time commitment while still earning meaningful rewards, reducing intimidation factor.
  • Persistent Data Creates Ownership Users immediately connected with progress that persisted across sessions. Seeing cumulative XP and maintaining streaks created emotional investment.
  • Visual Feedback is Essential Every interaction needed visual response: hover states, animations, progress bars, confetti. Silent interactions felt broken to users.
  • Positive Framing Over Locked Content Showing all achievements in full color (rather than grayed out) proved more motivating. Users felt inspired by goals rather than discouraged by locked content. Green completion states provide clear positive reinforcement.
  • Starting from Zero Matters Beginning at Level 0 with an empty progress circle gives users immediate gratification when they complete their first session and see measurable progress. Starting at Level 8 would diminish the sense of achievement.
